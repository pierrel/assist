* Title
=SandboxBackend.grep= crashes with =ValueError= when container exec
fails (parses OCI runtime error text as grep output)

* Summary
=SandboxBackend.grep= in =deepagents/backends/sandbox.py= blindly parses
each output line as =path:line:text= and does =int(parts[1])= to extract
the line number.  When the underlying =container.exec_run= fails to
start the process (chdir error, missing binary, container died mid-exec),
Docker returns the OCI runtime error message as stdout — a string that
/also/ contains colons.  The parser splits it, =int(' exec failed')=
raises =ValueError=, and the entire tool node crashes.

* Repro
Any condition that makes Docker exec fail at the runtime layer (before
the shell even starts) triggers this.  Easiest repro: a
=DockerSandboxBackend= with =work_dir== pointing at a directory that
doesn't exist inside the container.

#+begin_src python
from deepagents.backends.sandbox import SandboxBackend
# container exec runs with workdir="/workspace/references"
# but that dir doesn't exist inside the container
backend = SandboxBackend(container=..., work_dir="/workspace/references")
result = backend.grep("TODO", path="/")
# ValueError: invalid literal for int() with base 10: ' exec failed'
#+end_src

* Actual error trace (real prod crash)
#+begin_example
File ".../deepagents/backends/sandbox.py", line 781, in grep
    "line": int(parts[1]),
ValueError: invalid literal for int() with base 10: ' exec failed'
#+end_example

The actual stdout the parser was fed:

#+begin_example
OCI runtime exec failed: exec failed: unable to start container process: chdir to cwd ("/workspace/references"): no such file or directory: unknown
#+end_example

Splits on =:= (maxsplit=2) →
- =parts[0]= = ="OCI runtime exec failed"=
- =parts[1]= = =" exec failed"=
- =parts[2]= = =" unable to start..."=

=len(parts) >= 3= passes the existing guard.  =int(parts[1])= then
explodes.

* Suggested fix
In =deepagents/backends/sandbox.py:776-784=, guard the int parse:

#+begin_src python
for line in output.split("\n"):
    parts = line.split(":", 2)
    if len(parts) >= 3 and parts[1].strip().isdigit():
        matches.append({
            "path": parts[0],
            "line": int(parts[1]),
            "text": parts[2],
        })
#+end_src

Or — stricter — check the exec exit code via =result.exit_code= and bail
to =GrepResult(error=output, matches=None)= if non-zero before even
attempting to parse.  The current =2>/dev/null || true= suppresses
/shell-level/ errors but doesn't catch container-runtime-level failures.

* Workaround (currently shipped on this side)
Override =grep= with a =try/except ValueError=, return
=GrepResult(error=str(e), matches=None)=.  Implementation:
=assist/sandbox.py:232-263= on the
=ensure-references-dir-on-sandbox-start= branch (PR #103, commit
=0e3002c=).
