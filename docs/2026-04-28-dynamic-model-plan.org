** Design revision: callback handler, not middleware (2026-04-28)
The original plan proposed a =ModelNotFoundMiddleware=
(=AgentMiddleware= subclass with =wrap_model_call= /
=awrap_model_call=).  Reviewer feedback during iteration: the test
file "doesn't look like middleware" — and on inspection, the user is
right.  Middleware in this codebase *modifies* tool/model behavior
(rewrite errors, retry, evict, block).  What we actually want is a
*passive observer* that fires =invalidate_config_cache()= and
otherwise stays out of the way.

That's a =BaseCallbackHandler= subclass, not an =AgentMiddleware=.
Updated design:

#+begin_src python
# in assist/model_manager.py — colocated with the cache
class _ModelNotFoundCacheBuster(BaseCallbackHandler):
    def on_llm_error(self, error, **kwargs):
        if isinstance(error, openai.NotFoundError) and \
           getattr(error, "code", None) == "model_not_found":
            invalidate_config_cache()
#+end_src

Wired into =_build_openai_chat_model= via
=kwargs["callbacks"] = [_ModelNotFoundCacheBuster()]=.  Wins:
- Lives next to the cache it invalidates — single file, one concept.
- No =agent.py= changes; no wiring across three factories; no
  subagent inheritance gotcha.
- Tests live next to other cache tests in
  =tests/model_manager/test_dynamic_model.py= — no separate
  middleware test file.
- Smaller diff.

** Lazy =ThreadManager.model= (2026-04-28)
Important constraint surfaced during plan review: the user starts the
web server and vLLM in parallel; the web server must boot even when
vLLM is unreachable.  Today this works because =ChatOpenAI(...)=
construction is lazy (no network at init); the first =invoke()= is
where vLLM is touched.

The proposed probe in =_get_config()= would *break* that — calling
=select_chat_model(0.1)= during =ThreadManager.__init__=
(=assist/thread.py:170=, fired at module load via =manage/web.py:29=
=MANAGER = ThreadManager(ROOT)=) would HTTP the vLLM endpoint at
server startup.

Fix: make =ThreadManager.model= a lazily-resolved attribute.  Smallest
diff is a =@property= that calls =select_chat_model(0.1)= on first
access and caches on the instance.  First access happens inside
request handlers (=Thread.__init__=, =_initialize_thread=), so the
probe fires when the first request arrives, not at boot.

#+begin_src python
class ThreadManager:
    def __init__(self, root_dir: str | None = None):
        # ... existing setup, but DROP `self.model = select_chat_model(0.1)` ...
        self._model = None

    @property
    def model(self):
        if self._model is None:
            self._model = select_chat_model(0.1)
        return self._model
#+end_src

* Goal
Probe the LLM endpoint at first use, cache the discovered model id and
context length, and refresh the cache on a "model not found" error.
Drop the now-redundant environment variables.

The only env var that survives in the model-config layer is
=ASSIST_MODEL_URL= (the endpoint base).  =ASSIST_API_KEY= becomes
optional with a sensible local-vLLM default; =ASSIST_MODEL_NAME=,
=ASSIST_CONTEXT_LEN=, and =ASSIST_TEST_URL_PATH= go away entirely.

Non-goals:
- Multi-model selection or routing.  We have one model.
- Auto-retry of the failing request after cache refresh.  Refresh
  happens on the *next* request — mid-flight retry is more complex
  than warranted (see "Open questions" below).
- Public-OpenAI parity for context length.  Real OpenAI's =/v1/models=
  doesn't include =max_model_len=.  If anyone ever points this at real
  OpenAI we'll need a default fallback then; not now.

* Approach
A small probe helper in =assist/model_manager.py= replaces
=_load_custom_openai_config=.  Cache layout:

#+begin_src python
@dataclass(frozen=True)
class OpenAIConfig:
    url: str
    model: str
    api_key: str
    context_len: int

_cached_config: Optional[OpenAIConfig] = None
_cache_lock = threading.Lock()

def _probe_endpoint(url: str, api_key: str) -> OpenAIConfig:
    """GET {url}/models, return config for the first model in the list."""
    ...

def _get_config() -> Optional[OpenAIConfig]:
    """Return cached config or probe.  Returns None if URL unset."""
    ...

def invalidate_config_cache() -> None:
    """Bust the cache.  Next _get_config() call re-probes."""
    ...
#+end_src

** Why a probe (not env vars)
- vLLM already serves the answer.  =/v1/models= returns =id= and
  =max_model_len= in one call, no extra plumbing needed.
- The serving config is already authoritative — keeping it duplicated
  in three places (=.dev.env=, =.deploy.env=, systemd unit) is the
  source of the bug.
- A probe collapses three sources of truth into one: the live endpoint.

** Why not =functools.lru_cache=
The current code uses =@lru_cache(maxsize=1)= on
=_load_custom_openai_config=.  =lru_cache= has =.cache_clear()= but
isn't particularly elegant for the "invalidate on remote error" path,
and it lives on the function object rather than as an explicit module
state.  A module-level =Optional[OpenAIConfig]= with a =threading.Lock=
makes the lifecycle obvious and gives us a clean
=invalidate_config_cache()= entry point that can be called from
anywhere.

** API key handling
=ChatOpenAI= requires *some* =api_key= (or it falls back to the
=OPENAI_API_KEY= env var).  For local vLLM the value is meaningless —
=sk-local-test-123= is a placeholder.

Strategy:
1. If =ASSIST_API_KEY= is set, use it (back-compat for any existing
   deploy that has it set; same as today).
2. Else if =OPENAI_API_KEY= is set (langchain's own default), let
   =ChatOpenAI= use that.
3. Else default to the constant string =EMPTY=.  vLLM accepts it; real
   OpenAI rejects it (correct, since you need a real key for OpenAI
   anyway).

This removes =ASSIST_API_KEY= from =.dev.env= without breaking
auth-required deployments.

** Cache invalidation on "model not found"
The OpenAI Python SDK raises =openai.NotFoundError= with =code=
=model_not_found= when the requested model is no longer served.
=langchain-openai= surfaces this as a =NotFoundError= during
=invoke=/=ainvoke=.

[Design revised — see "Design revision: callback handler" section
above.  Implementation is a =BaseCallbackHandler= subclass attached to
the =ChatOpenAI= instance via the =callbacks== kwarg, *not* an
=AgentMiddleware=.  The text below is preserved as historical context;
treat the callback-handler design as authoritative.]

Plan: a tiny =ModelNotFoundMiddleware= wrapping =wrap_model_call= /
=awrap_model_call= that catches =NotFoundError= with
=model_not_found= code, calls =invalidate_config_cache()=, and
re-raises.  The caller sees the error; next request re-probes and gets
the new model.  No mid-flight retry — the request that triggered the
mismatch fails, but the system self-heals on the next request.

This is the same pattern as =bad_request_mw= in =assist/agent.py= —
catch a specific upstream error, do a side effect, surface the error.

Open question: do we want a one-shot in-flight retry?  See "Open
questions" below.

* Risks and gotchas
- *Probe at import time vs first use*: probe at first call to
  =select_chat_model=, not at module import.  Module import happens
  during test collection and we don't want the test runner trying to
  reach the vLLM endpoint.
- *httpx vs requests vs urllib*: prefer =httpx= (already in deps via
  =langchain_openai=).  No new dependency.
- *Threading*: =threading.Lock= around the cache is enough — single
  process, multiple agent threads in the web server.  No need for
  asyncio locks.
- *vLLM endpoint flake at startup*: if the probe fails the first time,
  raise a clear =RuntimeError= that says "Could not reach
  =${ASSIST_MODEL_URL}/models=; check that vLLM is running."  Don't
  silently fall back to a default — we want loud failure so the
  operator notices.
- *Stale cache during a model swap*: this is the behavior the
  middleware fixes.  Worst case: the request that catches the swap
  fails with =NotFoundError=; the user sees an error, retries, and
  the second request succeeds with the new model.  Acceptable.
- *=NotFoundError= shape*: =openai.NotFoundError= is the SDK exception;
  =code= is on the response body, not the exception attribute
  directly.  Check =error.body.get("error", {}).get("code")= or the
  =message= text.  Confirm during implementation.
- *Subagents inherit the middleware list selectively*: per the
  pattern documented in =write_collision.py=, only some subagent
  factories get the full list.  =model_not_found_mw= should go on
  *all* agent factories that make model calls — i.e. all of them.
- *=conversation_history/= and =docs/2026-04-27-qwen-quantization-guide.md=
  reference the removed vars*: history files are append-only, leave
  alone.  The quantization guide has =export ASSIST_MODEL_NAME=...=
  examples — update those to remove =MODEL_NAME= and =CONTEXT_LEN=
  exports, keep =MODEL_URL=.

* Open questions for review
1. *Mid-flight retry on model_not_found?*  Currently planned: no.  The
   alternative is to invalidate, re-probe, and resubmit the same
   request inside the middleware.  This is more complex (need to
   re-bind the agent's underlying ChatOpenAI to the new model) but
   gives a smoother UX.  Recommendation: ship without mid-flight
   retry, add later if it's actually painful.
2. *Should we probe eagerly at web-server startup?*  Pro: fail fast
   if vLLM is unreachable.  Con: slows down test runs that import the
   server module.  Recommendation: lazy on first use, but expose a
   =warm_cache()= helper that the web server's startup hook can call
   if it wants eager validation.
3. *Public-OpenAI fallback?*  Resolved during deployment-safety
   review: the existing fallback branch at =model_manager.py:102=
   (=_build_openai_chat_model(model, ...)=) is *already broken* —
   =model= is undefined at that scope (NameError).  The branch never
   runs in practice because every deploy has =ASSIST_MODEL_URL= set.
   Decision: *remove* the dead branch, replace with a clear
   =RuntimeError= ("=ASSIST_MODEL_URL= is required").  We don't want
   a silent fallback to public OpenAI; we want a loud failure if the
   local endpoint isn't configured.
4. *Config-cache TTL?*  Currently: cache is process-lifetime, only
   busted by =invalidate_config_cache()=.  Alternative: re-probe every
   N minutes.  Recommendation: skip the TTL — it adds complexity
   without solving a real problem (model swaps are rare and the
   error-driven invalidation handles them).
